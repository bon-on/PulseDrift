# 골든 룰

1. 코드 수정 전에 항상 활성 기능 명세부터 읽는다.
2. 연결된 ADR이 있으면 그 결정을 따른다. 충돌 시 조용히 덮어쓰지 말고 에스컬레이션한다.
3. 계획, 검증 증빙, 리뷰 메모는 저장소 안의 문서로 남긴다.
4. 치명적 요구사항이나 검증 증빙이 없으면 완료 처리하지 않는다.
5. 넓고 추측성인 리팩터링보다 좁고 되돌리기 쉬운 변경을 선호한다.
6. 명세가 예외를 허용하지 않는 한 iOS와 Android는 하나의 공유 게임플레이 구현을 유지한다.
7. 조작, 페이싱, 표현을 바꿔도 3레인 회피 핵심은 유지한다.
8. 채팅에만 존재하는 기능 작업을 추가하지 않는다. 사소하지 않은 변경은 반드시 명세 업데이트와 함께 간다.

## 제안 프로필 팩 오버레이

현재 승인된 프로필 팩 오버레이는 없다.
